What is the A* Algorithm?

The A* (A-star) algorithm for undirected graph is a popular method for finding the shortest path between two points. It combines the benefits of Dijkstra's Algorithm and Best-First Search to efficiently navigate through graphs and maps.

What it does: A* helps in finding the shortest route from a starting point to a destination.

How it works:

  • It uses a special formula that considers both the distance you've already traveled and an estimate of the remaining distance to your destination.
  • Step 1: Begin at your starting point and look at all the places you can go next.
  • Step 2: For each option, calculate the travel cost, which includes the distance covered so far plus an estimated distance to the goal (physical distance in graph).
  • Step 3: Choose the option with the lowest cost and move there.
  • Step 4: Repeat this process until you reach your destination.
  • This way, A* keeps you on track toward the shortest path by balancing the distance traveled and the distance left to cover.

What is Greedy Best-First Search (GBFS) Algorithm?

The Greedy Best-First Search (GBFS) algorithm for undirected graph is a pathfinding algorithm that focuses on exploring the most promising path based on a heuristic, which estimates the phyiscal distance to the goal.

What it does: GBFS searches for a path from a start node to an end node by always expanding the node that appears to be closest to the goal.

How it works:

  • It uses a heuristic to prioritize which nodes to explore next, based on their estimated distance to the goal.
  • Step 1: Start from the initial point and look at all the possible points you can move to next.
  • Step 2: Calculate the estimated physical distance (heuristic) from each point to the goal.
  • Step 3: Choose the point with the lowest estimated distance and move to that point.
  • Step 4: Repeat the process from the new point until you reach the goal or run out of points to explore.
  • This way, GBFS focuses on exploring paths that seem to lead directly to the goal, although it doesn't always guarantee the shortest path.

What is Dijkstra's Algorithm?

Dijkstra's algorithm for undirected graph is a famous algorithm used for finding the shortest paths between nodes in a graph. It works by exploring all possible paths and choosing the one with the lowest total distance.

What it does: Dijkstra's algorithm searches for the shortest path from a start node to an end node, ensuring the least total distance traveled.

How it works:

  • It systematically explores all possible paths from the start node to other nodes in the graph.
  • Step 1: Start from the initial point and set the distance to 0. All other nodes are set to infinity distance.
  • Step 2: Select the node with the smallest known distance and explore all its neighboring nodes.
  • Step 3: For each neighboring node, calculate the total distance from the start node. If this distance is smaller than the currently known distance, update the shortest distance.
  • Step 4: Repeat the process with the next node with the smallest known distance.
  • This way, Dijkstra's algorithm ensures that it finds the shortest possible path from the start node to the end node by always choosing the path with the least distance.

What is Breadth-First Search (BFS) Algorithm?

The Breadth-First Search (BFS) algorithm for undirected graph is a simple and systematic way of exploring nodes in a graph, layer by layer, starting from a given start node.

What it does: BFS searches for a path from a start node to an end node by exploring all nodes at the present depth level before moving on to nodes at the next depth level.

How it works:

  • It uses a queue to keep track of the nodes to be explored next.
  • Step 1: Start from the initial point and add it to the queue.
  • Step 2: Remove the first node from the queue and explore all its neighboring nodes.
  • Step 3: For each neighboring node, if it has not been visited yet, mark it as visited and add it to the queue.
  • Step 4: Repeat the process until the end node is found or the queue is empty.
  • This way, BFS ensures that it explores all possible paths from the start node in a systematic manner, layer by layer, until it finds the shortest path to the end node.

What is Depth-First Search (DFS) Algorithm?

The Depth-First Search (DFS) algorithm for undirected graph is a method for exploring a graph by starting at the root node and exploring as far as possible along each branch before backtracking.

What it does: DFS searches for a path from a start node to an end node by exploring as deep as possible down one path before backtracking and trying another path.

How it works:

  • It uses a stack to keep track of the nodes to be explored next.
  • Step 1: Start from the initial point and add it to the stack.
  • Step 2: Remove the last node from the stack and explore all its neighboring nodes.
  • Step 3: For each neighboring node, if it has not been visited yet, mark it as visited and add it to the stack.
  • Step 4: Repeat the process until the end node is found or the stack is empty.
  • This way, DFS explores one path as deep as possible before backtracking and exploring other paths, ensuring that all nodes are visited.

What is Kruskal's Algorithm?

Kruskal's algorithm is used to find the Minimum Spanning Tree (MST) in a graph, which connects all nodes with the least total link weight.

What it does: Kruskal's algorithm finds the subset of links that form a tree including every node, where the total weight of all the links in the tree is minimized.

How it works:

  • It sorts all the links in the graph by their weight in ascending order.
  • Step 1: Initialize an empty set to keep track of the links in the MST.
  • Step 2: Iterate through the sorted links and add each link to the MST if it doesn't form a cycle with the links already in the MST.
  • Step 3: Use a data structure called Union-Find to efficiently manage the connected components of the graph.
  • This way, Kruskal's algorithm ensures that the resulting tree has the minimum possible total link weight while connecting all the nodes in the graph.

Prim's algorithm

Use Prim's algorithm to find the Minimum Spanning Tree (MST) in a graph


What is Prim's Algorithm?

Prim's algorithm is used to find the Minimum Spanning Tree (MST) in a graph, which connects all nodes with the least total link weight.

What it does: Prim's algorithm finds a subset of links that form a tree including every node, where the total weight of all the links in the tree is minimized.

How it works:

  • It starts with an arbitrary node and grows the MST one link at a time.
  • Step 1: Initialize an empty set to keep track of the links in the MST and add the starting node to the set of visited nodes.
  • Step 2: Use a priority queue to keep track of the links with the smallest weight that connect the visited nodes to the unvisited nodes.
  • Step 3: Remove the link with the smallest weight from the priority queue, add it to the MST, and mark the connected node as visited.
  • Step 4: Repeat the process until all nodes are included in the MST or there are no more links to process.
  • This way, Prim's algorithm ensures that the resulting tree has the minimum possible total link weight while connecting all the nodes in the graph.

What is Fleury's Algorithm?

Fleury's algorithm for undirected graph is used to find Eulerian cycles and paths in a graph. An Eulerian path is a path that visits every link exactly once, while an Eulerian cycle is a cycle that visits every edge exactly once and returns to the starting node.

What it does: Fleury's algorithm finds an Eulerian cycle or path by traversing each link of the graph exactly once without repeating.

How it works:

  • It uses a strategy to avoid crossing bridges (links whose removal would disconnect the graph) until it is necessary.
  • Step 1: Start from a node with an odd degree (if all node don't have even degrees) if finding a path, or any node if finding a cycle.
  • Step 2: Choose a link to traverse next, prioritizing non-bridge links unless there is no alternative.
  • Step 3: Remove the chosen link from the graph and move to the next node.
  • Step 4: Repeat the process until all links have been traversed.
  • This way, Fleury's algorithm ensures that the resulting path or cycle visits every link exactly once, forming an Eulerian cycle or path.

What is Hierholzer's Algorithm?

Hierholzer's algorithm for undirected graph is used to find Eulerian cycles and paths in a graph. An Eulerian path is a path that visits every link exactly once, while an Eulerian cycle is a cycle that visits every link exactly once and returns to the starting node.

What it does: Hierholzer's algorithm finds an Eulerian cycle or path by traversing each link of the graph exactly once without repeating.

How it works:

  • It starts with an arbitrary node and forms a cycle by following unused links until it returns to the starting node.
  • Step 1: Start from any node and follow a path of unused links until returning to the start node, forming a cycle.
  • Step 2: If there are unused links left, pick any node in the current cycle with unused links and form another cycle.
  • Step 3: Combine this new cycle with the existing cycle by splicing it into the current cycle.
  • Step 4: Repeat the process until all links have been used.
  • This way, Hierholzer's algorithm ensures that the resulting path or cycle visits every link exactly once, forming an Eulerian cycle or path.

What is the Bellman-Ford Algorithm?

The Bellman-Ford algorithm for directed graph used to find the shortest path from a starting node to an ending node in a graph, even if the graph has links with negative weights.

What it does: The Bellman-Ford algorithm calculates the shortest path from a start node to an end node by iteratively relaxing the links of the graph.

How it works:

  • It initializes the distance to the start node to 0 and all other nodes to infinity.
  • Step 1: For each edge in the graph, update the shortest known distance to the target node if a shorter path is found through the current link.
  • Step 2: Repeat this process for a total of V-1 times, where V is the number of nodes in the graph.
  • Step 3: Check for negative-weight cycles by iterating through all links one more time. If a shorter path is found, it indicates the presence of a negative-weight cycle.
  • This way, the Bellman-Ford algorithm can handle graphs with negative weights and detect negative-weight cycles.

What are Nearest Neighbor and Sorted Edges Algorithms?

The Nearest Neighbor and Sorted Edges algorithms for undirected graphs are heuristics for solving the Traveling Salesman Problem (TSP). Algorithms aim to find the shortest route that visits each city once and returns to the start.

Traveling Salesman Problem (TSP): Given a list of cities and distances, find the shortest route that visits each city once and returns to the start.

Nearest Neighbor Algorithm: Visits the nearest unvisited node until all nodes are visited.

How it works:

  • Start at a given node.
  • Step 1: Mark the current node as visited.
  • Step 2: Move to the nearest unvisited node.
  • Step 3: Repeat until all nodes are visited.
  • Step 4: Return to the start to complete the cycle.

Sorted Edges Algorithm: Selects edges in increasing order of weight, avoiding cycles until the tour is complete.

How it works:

  • Sort all edges by weight.
  • Step 1: Initialize an empty tour set.
  • Step 2: Add edges to the tour if they don't form a cycle, except to complete the tour.
  • Step 3: Track connected components to avoid cycles.
